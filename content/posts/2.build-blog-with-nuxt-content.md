# 🚀 使用 NuxtJS Content 搭建博客

一篇简单说说搭建博客的过程。



前前后后包括部署大概花了快一周，由于 Nuxt Content 还是比较新的框架，为了能正确用到其中的 API ，文档都快翻烂了；同样 Nuxt 3 和 TailwindCSS 其实也没有多少接触啦，也得慢慢熟悉。总体来说这次搭建过程还是能学到不少。

## 技术栈的选择

看了大大小小的博客（包括很多已经因为域名服务器过期的 404 博客），好像目前比较流行的还是 Hexo / WordPress / Vuepress，甚至还有一个超级方便的工具 [Gridea](https://gridea.dev/) （感觉对喜欢一步到位并且不太想慢慢折腾的朋友会非常方便）。这些都是非常成熟的博客框架了，但还是没挑到想用的主题并且由不想在一个并不熟悉的框架下慢慢重写一个主题。在无数下安装了 hexo 后又闲置掉整个仓库之中不断循环。

直到逛着逛着看到 sukka 大佬的一篇 [优化博客的累计布局偏移（CLS）问题 | Sukka's Blog (skk.moe)](https://blog.skk.moe/post/fix-blog-cls/) 以及 [聊聊博客近期的更新 | Blog @ GIUEM](https://www.giuem.com/blog/recent-updates-of-my-blog-2021) 发现最近好像都比较流行使用使用 Gatsby / NextJS / … 这些框架重构 Hexo，但是我的技术栈并不太熟悉 React ，主要还是 Vue，所幸相同性质的 Vue 中也有 **NuxtJS**，并且也 [Nuxt 3](https://v3.nuxtjs.org/guide/concepts/introduction) 好像也比较成熟了，正好趁这个机会认真研究一下 NuxtJS 吧。

最后再搜着搜着 Nuxt Blog 相关的仓库时，发现了 **Nuxt Content** ✨，正好有我这个懒人所需要的一切玩意——「能够基于 Vue 组件文件和 Markdown 文件生成路由和导航，内容索引；Markdown 目录生成；基于 Vue 组件渲染 Markdown 内容；原生支持 **Shiki** 的代码高亮；静态站点的配置」（翻译自 Nuxt Content 仓库的 Features） ，能不能称得上是一个优秀的博客搭建的工具呢 😬。

最后为了更加方便样式开发，还是选择了一个 CSS 的框架 TailwindCSS，主要还是比较心水简洁现代的样式，花里胡哨的暂时还搞不来。



## 搭建的过程

由于不太熟悉 NuxtJS，直接在 Nuxt Content 的一个完整的 Demo [Atinux/content-wind](https://github.com/Atinux/content-wind)  的基础上修改的。

主要还是要不断翻文档[Nuxt Content (nuxtjs.org)](https://content.nuxtjs.org/guide/writing/content-directory) 。很多 API 和 feature 在文档中都有很比较详细的介绍。再不济就要去看 Nuxt 的文档啦。

～ 更加详细的过程还是看看后续有比较有趣的内容在写吧，因为目前只完成了一个基本的框架，感觉也没有什么好写的。🤥

～ 目前搭建的步骤更新在了 [about](/about) 页面。



##  相关问题

### 部署

`nuxt generate` 后生成了静态文件 `dist` 目录后，push 到 Github 的 gh-pages 分支后，使用 GitHub Pages 部署。然而打开后，发现各种样式都因为读取不了资源文件，完全就是「毛胚」状态的古早风格页面。一开始还以为 Nuxt 3 版本的静态 build 有问题（查了文档确实是要在 `nuxt.config.ts` 中添加 `target: 'static'`）。 

但是 deploy 了 n 遍终于知道，😵‍💫 Github Page 自带的 Page 主题引擎 jekyll 在部署页面的时候接管了渲染工作，会忽略 `_` 开头的文件，然而这里根本不需要 jekyll，所以在每次 push 到 `gh-pages` 分支时，带上一个 `.nojekyll` 禁用掉 jekyll 的空文件就好了。
