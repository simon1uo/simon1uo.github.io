# 🚀 使用 NuxtJS Content 搭建博客

这是一篇简单说说搭建博客的过程的文章，也是不会给你带来完整的搭建指导，但是也许其中的一些思路可以参考 🥲。

搭建这个博客前前后后，包括部署大概花了快一周，由于 Nuxt Content （最近升级到了 V2 支持 Nuxt 3）还是比较新的框架，为了能正确用到其中的 API 和特性 ，文档都快翻烂了；同样 Nuxt 3 和 TailwindCSS 其实也没有多少接触，也得慢慢熟悉。总体来说这次搭建过程还是能学到不少。

## 技术栈的选择

看了大大小小的博客（包括很多已经因为域名服务器过期的 404 博客），目前比较流行的还是 Hexo / WordPress / Vuepress，甚至还有一个超级方便的工具 [Gridea](https://gridea.dev/) （感觉对喜欢一步到位并且不太想慢慢折腾的朋友会非常方便）。这些都是非常成熟的博客框架了，但还是没挑到想用的主题并且由不想在一个并不熟悉的框架下慢慢重写一个主题。VuePress 其实也是个不错的选择，我搭建笔记使用了它，感觉它还是比较适合搭建知识库或者文档类的网站。

直到逛着逛着看到 sukka 大佬的一篇 [优化博客的累计布局偏移（CLS）问题 | Sukka's Blog (skk.moe)](https://blog.skk.moe/post/fix-blog-cls/) 以及 [聊聊博客近期的更新 | Blog @ GIUEM](https://www.giuem.com/blog/recent-updates-of-my-blog-2021) 发现最近好像都比较流行使用使用 Gatsby / NextJS / … 这些框架重构 Hexo，但是在我的技术栈中并不太熟悉 React ，目前主要还是 Vue 全家桶为主，所幸相同性质的 Vue 中也有 **NuxtJS**，并且也 [Nuxt 3](https://v3.nuxtjs.org/guide/concepts/introduction) 好像也比较成熟了，正好趁这个机会认真研究一下 NuxtJS 吧。

最后再搜着搜着 Nuxt Blog 相关的仓库时，发现了 **Nuxt Content** ✨，正好有我这个懒人所需要的一切玩意——「能够基于 Vue 组件文件和 Markdown 文件生成路由和导航，内容索引；Markdown 目录生成；基于 Vue 组件渲染 Markdown 内容；原生支持 **Shiki** 的代码高亮；静态站点的配置」（翻译自 Nuxt Content 仓库的 Features） ，感觉这就是我要找的东西了，并且搭建起来也挺简单 😬。

最后为了更加方便样式开发，还是选择了一个 CSS 的框架 TailwindCSS，主要还是比较心水简洁现代的样式，花里胡哨的暂时还搞不来。

## 发现问题

这部分记载一下搭建遇到的一些问题。很傻很呆。

#怎么就点了一下导航栏然后突然啥都没了！？😵

当点进某篇文章，然后从导航栏点击返回首页或者归档的页面时，会出现空白。一开始还以为没啥，只是本地开发的问题，后来看了看错报错，是 **插入节点的问题**。首先排查了一下是不是导航栏的问题，毕竟依赖的是 `<NuxtLink>` 这个标签实现的，但改来改去问题依旧没有解决。搜了搜网上还真没找到几个解决方案，后来加上 nuxt 关键字，找到了一个 `Nuxt` 仓库的 issue ，果然 Vue 3 的新特性真不该乱用，在 `[...slug].vue` 这个页面组件中，我过于超前采用了「不带根标签」写模版的方式，偏偏就是这个小小的动作。

> 参考： [Failed to execute 'insertBefore' on 'Node': The node before which the new node is to be inserted is not a child of this node. · Issue #1664 · nuxt/framework (github.com)](https://github.com/nuxt/framework/issues/1664)



#静态文件是生成了/页面也有/怎么样式不见了🤦🏻‍♂️？

`nuxt generate` 后生成了静态文件 `dist` 目录后，push 到 Github 的 gh-pages 分支后，使用 GitHub Pages 部署。然而打开后，发现各种样式都因为读取不了资源文件，完全就是「毛胚」状态的古早风格页面。一开始还以为 Nuxt 3 版本的静态 build 有问题（查了文档确实是要在 `nuxt.config.ts` 中添加 `target: 'static'`）。 

但是 deploy 了 n 遍终于知道，😵‍💫 Github Page 自带的 Page 主题引擎 jekyll 在部署页面的时候接管了渲染工作，会忽略 `_` 开头的文件，然而这里根本不需要 jekyll，所以在每次 push 到 `gh-pages` 分支时，带上一个 `.nojekyll` 禁用掉 jekyll 的空文件就好了。

> 参考：[Nuxt - GitHub Pages (nuxtjs.org)](https://nuxtjs.org/deployments/github-pages/) / [nuxt.js - Nuxt3 - `_nuxt`-directory not found (404) on GitHub Pages - Stack Overflow](https://stackoverflow.com/questions/71951915/nuxt3-nuxt-directory-not-found-404-on-github-pages?noredirect=1&lq=1) / [jekyll - Pushed .nojekyll file to Github pages, no effect? - Stack Overflow](https://stackoverflow.com/questions/47356997/pushed-nojekyll-file-to-github-pages-no-effect) / [About GitHub Pages - GitHub Docs](https://docs.github.com/cn/pages/getting-started-with-github-pages/about-github-pages#static-site-generators)
